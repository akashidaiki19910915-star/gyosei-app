# 運用手順・事故防止ガイド

## 1. 日常運用フロー
- 顧客登録
- 許認可ヒアリング
- 必要書類・タスク生成
- 概算見積
- 正式見積
- 受注
- 案件化
- 請求書作成
- 売上登録
- 入金確認
- 経費登録
- CSV/Excel出力
- バックアップ

## 2. 行政書士実務上の注意
- 業際確認
- 委任状・本人確認
- 申請先・期限確認
- 必要書類の最新版確認
- 顧客向け帳票と内部情報の分離
- 内部加算明細を客先提出用に出さない
- 値引き行は表示してよい
- 相談記録・対応履歴を残す

## 3. 請求・売上・経費の注意
- 見積金額、請求金額、売上金額の一致確認
- 一部入金・未入金の管理
- 実費・法定費用と報酬の分離
- 消費税10%表示
- インボイス制度対応が必要な場合の確認事項
- 電子取引データ保存の注意

## 4. 電子帳簿保存法・証憑管理
- 電子で受領/送付した見積書、請求書、領収書、契約書等は保存対象になり得る
- 日付、金額、取引先で検索できる形を意識する
- 原本PDF/メール/添付ファイルの保存はこのアプリだけに依存しない
- 正式な保存要件は国税庁情報を確認する

## 5. 個人情報・守秘管理
- 顧客情報、在留資格、許認可資料、本人確認資料は機微情報として扱う
- 不要なデータは入れない
- テストデータと本番データを混ぜない
- バックアップファイルの保管場所に注意する
- 共有URLや端末紛失時のリスクに注意する

## 6. バックアップ・復元
- 作業前バックアップ
- 大量取込前バックアップ
- 復元前確認
- 復元後の件数確認
- テストデータ削除確認

## 7. 事故防止チェックリスト
- 見積前
- 受注前
- 案件化前
- 請求前
- 入金確認前
- 申請前
- 納品前

## 8. やってはいけない運用
- ヒアリング結果から直接案件化しない
- 内部加算明細を客先提出しない
- DB未定義項目を保存しようとしない
- 本番環境でテストデータを放置しない
- バックアップなしで大量取込しない
- 申請先の最新様式確認を省略しない

## 9. 今後の実装検討事項
実装はしない。検討事項として記載のみ。
- 操作ログ
- バックアップ忘れ警告
- 証憑ファイル添付
- インボイス登録番号欄
- 電帳法対応チェック欄
- 個人情報削除/マスキング手順


## 10. 実務投入前に毎回確認すること
- GitHub Pages を **Ctrl+F5**（macOS: Cmd+Shift+R）で更新し、最新版JSが反映されること。
- Console に重大な error がないこと。
- 業務フロー（許認可ヒアリング→概算見積→正式見積→受注→案件化→請求書作成→売上登録→入金登録）が通ること。
- 見積書・請求書に内部加算明細 / `addon_breakdown` / 内部メモ / 内部査定が表示されないこと（値引き行は表示されること）。
- 同一見積から案件・請求・売上が重複作成されないこと。
- CSV/Excel取込で余計な列があってもDB保存エラーにならないこと。
- バックアップ復元時に未定義カラムが送信されないこと。
- 顧客・案件・見積・売上の削除時に関連データが不自然に残らないこと。
