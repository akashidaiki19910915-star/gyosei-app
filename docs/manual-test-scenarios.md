# 手動テストシナリオ

本ドキュメントは、PR #202〜#212 で追加・修正された一連の業務フロー（許認可ヒアリング、概算見積、正式見積、案件化、請求、売上、CSV/Excel取込、バックアップ復元）を、**最短で動作確認するための実機テスト観点**を整理したものです。

- 目的：本番運用前の手動確認を短時間で漏れなく実施する
- 前提：既存マスタ（顧客、案件、見積、請求、売上、経費、取込機能、バックアップ復元機能）が利用可能
- 非対象：新機能追加、DBスキーマ変更、RLSやservice_role設定変更

---

## 2. 概算見積 → 正式見積

確認項目：
- 基本報酬
- 加算
- 値引き
- 実費
- 消費税
- 合計
- 客先提出用見積に内部加算明細が出ないこと

最短テスト手順：
1. 上記 1 のいずれか（例：古物商許可）で概算見積を作成。
2. 基本報酬、加算、値引き（例：-5,000円）、実費を入力。
3. 正式見積へ変換。
4. 金額計算（税抜・税込・合計）の整合性を確認。
5. 客先提出用見積プレビュー／出力で、内部用の加算内訳が非表示であることを確認。

期待結果：
- 概算→正式変換後も金額構成が崩れない。
- 内部管理用項目は客先表示に出ない。

## 3. 正式見積 → 受注 → 案件化

確認項目：
- 受注ステータスで案件化できる
- 同一見積から案件が重複作成されない
- 失注見積から案件化する場合は確認が出る
- 案件金額と見積金額が一致する

最短テスト手順：
1. 正式見積のステータスを「受注」に変更。
2. 案件化を実行し、案件が1件作成されることを確認。
3. 同じ見積で再度案件化を試し、重複作成されないことを確認。
4. 別見積を「失注」にして案件化操作を行い、確認ダイアログの表示を確認。
5. 作成案件の金額が見積合計と一致することを確認。

## 4. 請求書作成

確認項目：
- 見積から請求書を作成できる
- 同一見積から請求書が重複作成されない
- 値引き行が請求書に表示される
- 内部加算明細は請求書に表示されない

最短テスト手順：
1. 正式見積から請求書作成を実行。
2. 請求書が1件生成されることを確認。
3. 同一見積から再作成を試行し、重複防止されることを確認。
4. 請求書表示で値引き行が明示されることを確認。
5. 内部加算の詳細が請求書に出ないことを確認。

---

## 5. 売上・入金管理

確認項目：
- 売上登録済み表示
- 未入金表示
- 一部入金表示
- 請求額・入金額の差異表示

最短テスト手順：
1. 作成済み請求書に対して売上登録を実行。
2. 入金未登録状態で「未入金」表示を確認。
3. 一部入金（例：請求 110,000円に対し 50,000円）を登録。
4. 「一部入金」表示と差異（60,000円）が表示されることを確認。

## 6. 案件進行監査

確認項目：
- 期限超過
- 期限接近
- 次回対応日超過
- 書類未回収あり
- 未完了タスクあり
- 請求未作成
- 売上未登録
- 入金未確認
- 問題なし表示

最短テスト手順：
1. 案件A：期限を過去日に設定し「期限超過」を確認。
2. 案件B：期限を近日期日に設定し「期限接近」を確認。
3. 案件C：次回対応日を過去日にして超過表示を確認。
4. 案件D：必要書類を一部未回収に設定。
5. 案件E：タスクを未完了で残す。
6. 案件F：受注済みだが請求未作成状態にする。
7. 案件G：請求あり・売上未登録を作る。
8. 案件H：売上あり・入金未確認を作る。
9. 案件I：すべて完了状態にして「問題なし」を確認。

## 7. CSV/Excel取込

確認項目：
- 顧客CSV
- 案件CSV
- 売上CSV
- 経費CSV
- 余計な列があっても400エラーにならない
- 空欄・日付・金額が正常処理される

最短テスト手順：
1. 顧客CSV取込（余計な列 `memo_extra` を追加）を実行。
2. 案件CSV取込で日付列（YYYY-MM-DD）と空欄列を混在させる。
3. 売上CSV取込で金額列に整数・カンマ入り数値を含める。
4. 経費CSV取込で空欄/0円/通常金額を混在させる。
5. 取込後、一覧で件数・主要項目・金額が期待通りであることを確認。

テストデータ例（抜粋）：
- 顧客CSV
  - `customer_name,customer_type,phone,memo_extra`
  - `株式会社テストA,法人,03-1111-2222,追加列`
- 案件CSV
  - `case_name,service_type,due_date,next_action_date`
  - `古物商許可A,古物商許可,2026-06-30,`
- 売上CSV
  - `sales_date,amount,description`
  - `2026-05-01,"110,000",古物商許可報酬`
- 経費CSV
  - `expense_date,amount,category`
  - `2026-05-02,0,交通費`

## 8. バックアップ復元

確認項目：
- 復元前確認ダイアログ
- 成功件数
- スキップ件数
- エラー件数
- 復元後に一覧が壊れない

最短テスト手順：
1. テスト用バックアップファイルを選択し復元操作を開始。
2. 復元前確認ダイアログが表示されることを確認。
3. 復元結果（成功/スキップ/エラー件数）表示を確認。
4. 顧客・案件・見積・請求・売上一覧を再表示し、表示崩れや参照切れがないことを確認。

## 9. 回帰確認

確認項目：
- 顧客管理
- 案件管理
- 見積作成
- 見積編集
- 見積書出力
- 請求書出力
- 案件化
- 売上管理
- 経費管理
- CSV/Excel出力
- Supabase保存

最短テスト手順（スモーク）：
1. 顧客を1件新規登録・更新・検索。
2. 案件を1件登録・ステータス更新。
3. 見積を作成・編集し、見積書出力。
4. 案件化・請求書作成・請求書出力。
5. 売上/経費を各1件登録し、一覧反映を確認。
6. CSV/Excel出力を実行し、ファイルが生成されることを確認。
7. 画面再読込後もデータ保持（Supabase保存）を確認。

## 実施メモ（推奨）
- 1シナリオごとに「入力」「期待値」「実績」「差分」「スクリーンショット有無」を記録する。
- 失敗時は、どの画面・どのボタン・どのデータで再現したかを残す。
- 可能なら、同一データで「概算→正式→案件化→請求→売上→入金」まで通し確認して整合性を検証する。
