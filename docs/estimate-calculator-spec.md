# 見積自動算出モジュール 実装仕様

## 目的

各案件について、業務種別・申請区分・顧客属性・難易度・実費・加算要素を入力することで、見積金額を自動算出し、既存の見積・案件・請求機能へ連携する。

本機能は、行政書士業務の報酬額を機械的に確定するものではなく、正式受任前の「概算見積」「正式見積」の作成補助を目的とする。

## 法務・実務上の前提

- 行政書士報酬は各事務所が自由に定める。
- ただし、事務所内で報酬額表の掲示義務がある。
- 同一業務でも、案件の難易度、確認資料の有無、提出先、依頼範囲、急ぎ対応などで報酬額が変動する。
- 自動算出結果は「確定報酬」ではなく、最終確認後に正式見積として確定する。
- 画面上に必ず「概算です。資料確認後に正式見積を提示します。」という注意書きを表示する。

- 役員・専技等の住民票等取得有無

## 算出ロジック

合計額 = 基本報酬 + 加算報酬 + 実費 - 値引き

消費税 = 報酬部分 × 税率

請求合計 = 報酬税込 + 実費

実費は原則として消費税計算対象外で管理する。

## DB拡張案

```sql
create table if not exists estimate_calculations (
  id uuid primary key default gen_random_uuid(),
  user_id uuid not null,
  estimate_id uuid,
  case_id uuid,
  client_id uuid,
  business_type text not null,
  application_type text,
  jurisdiction_type text,
  applicant_type text,
  urgency text,
  document_readiness text,
  input_data jsonb default '{}'::jsonb,
  base_fee bigint default 0,
  addon_fee bigint default 0,
  discount_amount bigint default 0,
  expense_amount bigint default 0,
  taxable_subtotal bigint default 0,
  tax bigint default 0,
  total bigint default 0,
  calculation_note text,
  source_note text,
  status text default '概算',
  created_at timestamptz default now(),
  updated_at timestamptz default now()
);

alter table estimate_calculations enable row level security;

drop policy if exists "estimate_calculations_own" on estimate_calculations;
create policy "estimate_calculations_own" on estimate_calculations
for all
using (auth.uid() = user_id)
with check (auth.uid() = user_id);
```

## 既存見積への連携

自動算出後、以下のボタンを出す。

- 見積に反映
- 案件に反映
- 見積書プレビュー
- 請求データ作成

見積明細へは以下のように分解して登録する。

1. 基本報酬
2. 難易度加算
3. 急ぎ対応加算
4. 代理取得・訪問加算
5. 値引き
6. 実費

## 表示注意文

見積自動算出画面に以下を表示する。

> この金額は入力内容に基づく概算です。正式な報酬額は、必要資料・申請先・業務範囲を確認したうえで確定します。実費・法定費用は変更となる場合があります。

## 実装時の注意

- 既存の app.js は大きいため、既存処理を壊さない。
- 可能なら estimate-calculator.js として分離し、index.html で読み込む。
- 最初はDOM追加を最小限にし、既存の見積フォームへ結果反映する方式にする。
- 保存時に二重登録しない。
- Supabase未接続でも画面上の概算計算だけは動作するようにする。
- node --check app.js を必ず実行する。
- ブラウザでサンプル1・サンプル2を手入力して結果を確認する。
