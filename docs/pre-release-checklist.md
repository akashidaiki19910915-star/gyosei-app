# 販売前・実務投入前チェックリスト（行政書士向け）

本チェックリストは、**新機能追加ではなく**、現時点の案件管理・見積請求・売上経費管理機能が初期実務投入可能かを確認するためのものです。

## 1. 実務導線チェック

以下の業務導線を上から順に確認する。

- [ ] 顧客登録
- [ ] 許認可ヒアリング
- [ ] 必要書類・タスク生成
- [ ] 概算見積
- [ ] 正式見積
- [ ] 受注
- [ ] 案件化
- [ ] 請求書作成
- [ ] 売上登録
- [ ] 入金管理
- [ ] 経費管理
- [ ] CSV/Excel出力
- [ ] バックアップ/復元

## 2. 帳票チェック

見積書・請求書の表示と金額整合性を確認する。

- [ ] 見積書に内部加算明細が出ない
- [ ] 請求書に内部加算明細が出ない
- [ ] 値引き行は表示される
- [ ] 顧客向け備考のみ表示される
- [ ] 消費税・合計が一致する

## 3. DB保存チェック

保存処理と運用ルール順守を確認する。

- [ ] 未定義カラムを送信しない
- [ ] Supabase保存で400エラーが出ない
- [ ] DBスキーマ変更なし
- [ ] RLS変更なし
- [ ] service_role未使用

## 4. 重複防止チェック

二重登録・重複生成が発生しないことを確認する。

- [ ] 同一見積から案件重複作成なし
- [ ] 同一見積から請求書重複作成時に確認あり
- [ ] 同一案件から売上重複登録なし
- [ ] CSV/Excel取込で重複が増えない

## 5. 売上・入金チェック

入金ステータスと再計算ロジックの整合性を確認する。

- [ ] 未入金
- [ ] 一部入金
- [ ] 入金済み
- [ ] 入金履歴削除後の再計算
- [ ] 売上金額と案件金額の不一致表示

## 6. 最低限の手動テスト手順（10分）

10分で全体確認できる最小順序を以下に示す。

1. 顧客登録→許認可ヒアリング→必要書類・タスク生成を実施する。  
   期待結果：顧客・案件準備情報が保存され、画面エラーが出ない。
2. 概算見積を作成後、正式見積へ確定する。  
   期待結果：金額・税額・合計が整合し、内部加算明細は顧客帳票に表示されない。
3. 正式見積から受注→案件化を実行する。  
   期待結果：同一見積から案件が重複作成されない。
4. 案件から請求書作成を実施する。重複作成を試行する。  
   期待結果：初回は正常作成され、重複作成時は確認が表示される。
5. 売上登録後、未入金→一部入金→入金済みまで更新する。入金履歴を1件削除する。  
   期待結果：ステータスと残額が再計算され、表示不整合がない。
6. 経費登録後、CSV/Excel出力を実行する。  
   期待結果：出力ファイルが生成され、重複データが増えない。
7. バックアップを取得し、復元を実施する。  
   期待結果：主要データ（顧客・案件・見積・請求・売上・入金・経費）が復元される。

## 7. 初期版完成判定

以下を**すべて満たした場合、初期実務投入可**と判定する。

- [ ] 赤エラーなし
- [ ] 保存エラーなし
- [ ] 帳票表示崩れなし
- [ ] 金額不一致なし
- [ ] 重複作成なし
- [ ] バックアップ復元確認済み
